# PTX ldmatrix 指令说明

## 1. 指令语法

```
ldmatrix.sync.aligned.shape.num{.trans}{.ss}.type  r, [p];
```

| 限定符 | 可选值 | 说明 |
|--------|--------|------|
| `.shape` | `.m8n8`, `.m16n16` | 矩阵形状（常用 .m8n8） |
| `.num` | `.x1`, `.x2`, `.x4` | 加载 1/2/4 个矩阵 |
| `.trans` | 可选 | 加载为列主序（转置） |
| `.ss` | `.shared`, `.shared::cta` | 数据来源（共享内存） |
| `.type` | `.b16`, `.b8` | 元素位宽 |

**注意**：`.sync` 和 `.aligned` 是强制限定符（warp 内所有线程必须执行相同指令）。

---

## 2. 参数说明

| 参数 | 说明 |
|------|------|
| `r` | 目标寄存器，花括号包裹的向量表达式，含 1/2/4 个 `.b32` 寄存器（与 `.num` 对应） |
| `[p]` | 源地址操作数（每个线程提供一个地址，指向共享内存中某一行的起始位置） |

**目标寄存器数量**：
- `.x1` → 1 个 `.b32` 寄存器 `{r0}`
- `.x2` → 2 个 `.b32` 寄存器 `{r0, r1}`
- `.x4` → 4 个 `.b32` 寄存器 `{r0, r1, r2, r3}`

---

## 3. 线程-地址映射（每个线程提供哪一行的地址）

加载 `.m8n8` 矩阵（8×8，每元素 16-bit，即每行 16 bytes）时，每个矩阵需要 8 个行地址，由 8 个线程提供：

| `.num` | Thread 0–7 | Thread 8–15 | Thread 16–23 | Thread 24–31 |
|--------|-----------|------------|-------------|-------------|
| `.x1` | 矩阵1 行0–7 的地址 | — | — | — |
| `.x2` | 矩阵1 行0–7 | 矩阵2 行0–7 | — | — |
| `.x4` | 矩阵1 行0–7 | 矩阵2 行0–7 | 矩阵3 行0–7 | 矩阵4 行0–7 |

- `thread i`（i=0..7）提供第 i 行的起始地址
- thread 8–31 的地址用于额外矩阵（`.x2`/`.x4`），或在 sm_75 以下需复制低位线程地址

**地址对齐要求**：每行地址须 16-byte 对齐（一组 4 个连续线程加载 16 bytes）。

---

## 4. 每个线程加载的元素位置（.m8n8 + .b16，行主序）

8×8 矩阵，每元素 16-bit，共 128 bytes = 16 bytes/行 × 8 行。

**一组 4 个连续线程**（thread 4k, 4k+1, 4k+2, 4k+3）共同加载第 k 行的 8 个元素：
- thread 4k+0 → 列 0, 1（存入寄存器低/高 16-bit）
- thread 4k+1 → 列 2, 3
- thread 4k+2 → 列 4, 5
- thread 4k+3 → 列 6, 7

即每个线程的 `.b32` 目标寄存器中存放**同一行的 2 个连续 16-bit 元素**。

```
行  r    = laneid / 4      // 即 laneid >> 2
列  c[0] = (laneid % 4) * 2
列  c[1] = (laneid % 4) * 2 + 1
```

**配合 mma.m16n8k16 使用时**（需要加载 A: 16×16 和 B: 16×8）：
- A 矩阵（16×16，f16）= 2 个 8×8 矩阵拼成：用 `.x2`（或两次 `.x1`）加载，各 8 行
- B 矩阵（16×8，f16）= 1 个 8×8 矩阵 `.trans`（转置）加载

---

## 5. `.trans` 的作用

不加 `.trans`：按**行主序**加载，每个线程得到某行的 2 个连续元素（适合 row-major 矩阵 A）。

加 `.trans`：按**列主序**加载，每个线程得到某列的 2 个连续元素（适合 col-major 矩阵 B，即 B^T 以行主序存储的情况）。

---

## 6. 代码示例

```ptx
// 加载 1 个 8×8 矩阵（行主序）
.reg .b64 addr;
.reg .b32 d;
ldmatrix.sync.aligned.m8n8.x1.shared::cta.b16 {d}, [addr];

// 加载 2 个 8×8 矩阵（列主序，即转置）
.reg .b64 addr;
.reg .b32 d<2>;
ldmatrix.sync.aligned.m8n8.x2.trans.shared.b16 {d0, d1}, [addr];

// 加载 4 个 8×8 矩阵（行主序）
.reg .b64 addr;
.reg .b32 d<4>;
ldmatrix.sync.aligned.m8n8.x4.b16 {d0, d1, d2, d3}, [addr];
```

---

## 7. 硬件要求

- 基础 `.m8n8` 形状：需要 **sm_75** 或更高（Turing+）
- `.m16n16` / `.m8n16` 形状：需要 sm_100a / sm_120a 等特定架构
- `.b8` 类型：需要 sm_100a 等

---

## 8. 与 mma.m16n8k16（f16）配合使用的典型流程

```
// 假设 A(16×16) 和 B^T(8×16) 存于 shared memory（行主序）
// 每个 thread 设置好地址指向对应行

// 加载 A: 2 个 8×8 矩阵（覆盖 A 的前 8 行 + 后 8 行）
ldmatrix.sync.aligned.m8n8.x4.shared::cta.b16 {Ra0, Ra1, Ra2, Ra3}, [addr_A];
// Ra0/Ra1 → A 的前 8 行（lane 0-7 提供地址）
// Ra2/Ra3 → A 的后 8 行（lane 8-15 提供地址）

// 加载 B: 1 个 8×8 矩阵转置（B 以列主序存储，或 B^T 以行主序存储）
ldmatrix.sync.aligned.m8n8.x2.trans.shared::cta.b16 {Rb0, Rb1}, [addr_B];

// 执行 mma
mma.sync.aligned.m16n8k16.row.col.f32.f16.f16.f32
    {Rd0, Rd1, Rd2, Rd3},
    {Ra0, Ra1, Ra2, Ra3},
    {Rb0, Rb1},
    {Rc0, Rc1, Rc2, Rc3};
```
